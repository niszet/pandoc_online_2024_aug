現時点で何が出来るのか？

  • テンプレートの恩恵を受けられる。
    • あらかじめ決められた順序で表示する場合に向いている。
    • 値がない場合も違和感なく非表示に出来る
    • 値を列挙するときの表示の仕方を制御できる
    • メタデータ/変数で値を与えることに意味がある場合に有効
    • フィルタを書くよりは敷居が低い。複雑さも少ない。
      • しかし細かい制御や応用はできない。
    • データと処理を分離出来る。
      • データはyamlに分離してフィルタで処理すればいいという話はある。
      • そもそもluaフィルタ使えばなんとでもなることが多い
  • 具体的な作例
    • 表紙ページを変更（著者名先頭にもってくるとか、並べるとか自由に出来る）
      • docxは固定順序であった。対応するメタデータが存在すると表示される。
    • 奥付の作成など(sky_y氏インスパイア)
  • 基本的な手順
    • 適宜修正したテンプレートに変数/メタデータで値を埋め込みファイルを作成
    • 最終的な出力を作る際に、出来上がったファイルを--include-before(after)-bodyで取り込み挿入する。

docxならではの事情

  • スタイルは文書全体で共有するため、テンプレートでは扱えない。

    • reference.docxで管理する。

  • セクションは扱える。セクションを使えばテンプレート内のページ設定を変えることは可能。

    • 縦/横
    • ページのサイズ・余白
    • ページ内上寄せ/下寄せ
    • 段組み変更
    • デフォルトの紙面の設定はreference.docx内にある。

  • 表紙のこと以外は raw記法を使えば今までもLua filterで出来た事ではある。 *　セクション挿入する、toc挿入する･･･などは出来る。

  • 既往のdocxファイルを(unzipしたのち）タグを適宜修正して挿入してうまくいくケースもあるかも。

現時点で何が出来ないのか？

  • Pandocには--to oxmlがない（大問題）
  • --to docxでTemplateを使うとdocument.xmlが不完全になる
    • 参照すべき他のxmlへのリンクがないなど）ので正しいdocxをつくれない（=MS Wordで開けない）
  • 出力形式を指定しないとHTMLとして出力される。本文に文字列が入っているとHTMLタグに囲われる
  • odt (OpenOffice text document)の opendocument のように、 docx にも openxml 出力形式が追加されればもう少し楽に出来るのだが･･･

Pandoc’s Template 導入編

Pandocのテンプレート機能

  • 変数を使い、テンプレート内に文字列を挿入するなどしてファイルを作ることが出来る。

    • Pandoc User’s Guide — Template

  • Pandoc単体で実行可能

    • 環境構築せずすぐ使える

  • Templateにはifやforなどがあるので色々出来そうだが出来ない。

  • 基本的なフォーマットが決まっていて値を置き換えるのに使う。

    • ifは値があるかないかチェックでの分岐
    • forは全ての要素それぞれで置換して列挙
    • サブテンプレートの埋め込み（Partials）
    • その他体裁を整える機能（Nesting、Breakable Spaces）
    • 変数の文字列に対する処理（Pipe）

  • 複雑な処理をしたければLua filterを使う。 ## テンプレートに値をセットする方法

  • データの種類としては変数（Variable）とメタデータがある。

  • メタデータは一般的に（？）MarkdownファイルのYAMLブロックに入れる事が多いが引数（-M/--metadata）でも与えられる。

  • テンプレートの場合は変数の方がメタデータより優先される。

    • この挙動についてはこちらの記事を参照 Pandocでメタデータはテンプレート変数になるかならないか - マクロツイーター
      • 変数はLuaフィルタから一応は見えるようになったが、値を変更してもPandocの挙動に影響なし=結論変わらず。
      • フィルタ自身がテンプレートを扱う場合の動作は未確認。

  • メタデータはメタデータファイルとしても与えることが出来る（--metadata-file）

  • 細かい話は実践編で。

デフォルトファイル(Defaults files)も使える

  • 使い回しがきく設定は、デフォルトファイルにまとめてしまう方がよい。
    • デフォルトファイルはメタデータや変数や設定を保存するためのファイル。
    • テンプレートとたぶん相性はよい。
  • 使い方は下記の記事を参照するのが良い。基本的な部分は変更ない。
    • [Pandoc 2.8 新機能] 長いオプションをファイルにまとめるDefault file #YAML - Qiita
  • 今回のケースでは値の設定は metadataかvariableに対してなので次のようにする。（トップ階層に置くのではない）
    • 詳細はマニュアルを。
metadata:
  title: default-metadata
variables:
  subtitle: default-variable

テンプレートの文法(変数)

  • 変数は$...$ か ${...} の書式で使う。たまにマニュアルに誤記がある。未指摘。
  • コメントは$--で入れる。 リテラル$ は　$$ と入力。
  • テンプレート変数は文字ではじまり、文字、数字、_、 -、 .を使える。日本語も使えた。
    • it, if, else, endif, for, sep,  endfor は使えない
    • 一部、Pandocが使用している変数名がある
      • 詳しくは Pandoc User’s Guide — Variables set automatically を参照。
  • . は構造化された変数（structured variable）に使う。
    • hoge.fugaみたいにしてアクセスする。
hoge:
  fuga: boo
  piyo: foo
  • 変数名に.を入れると上記の仕様からテンプレート側からアクセス出来なくなるので注意。
  • YAMLメタデータブロックでは末尾_の変数名は無視されるが、-Mや-Vは無視されない。
    • Pandoc User’s Guide　— Extension: yaml_metadata_block ## テンプレートの文法(if)
  • テンプレート中で$if(変数)$などで 条件分岐が使える。ただし、trueかfalseかだけを判定。比較演算子等はない。
    • $elseif(変数)や$else$もあるのである程度分岐は出来る。
      • tocがある場合のみtoc-titleが表示されるなどで使われている。
    • 詳しくは Pandoc User’s Guide — Conditionals
  • trueとなるものは以下の通り
    • 連想配列（map。key: valueのもの）
  • 配列（array, list）のうち、少なくともひとつのtrueの値をもつもの
    • listの中身が全て""ならfalse
  • 空文字""ではない文字列
  • booleanのtrue。-M titleのように与えるとtitleはbooleanのtrue扱い。
    • -M title=trueも同じ。-M title=false は booleanのfalse扱い。

テンプレートの文法(for)

  • $for(変数)$でループ
    • Pandoc User’s Guide — For loops
  • 単一の変数（文字列など）はその値のみを反映させて終わり
  • list(array)はそれぞれの値を繰り返し適用し
  • 連想配列(map)はloop内でそのkeyを呼ぶ形で使う。
author: 
  name: "MF2 niszet"

なら次のようにしてアクセス出来る（ということでいいのか？）

$for(author)$
  $it.name$
$endfor$
  • それぞれの値を代入したものが連続して出力されるのは同じ。

サブテンプレート（Partials）

  • 別のテンプレートファイルを使い、指定した場所に結果を挿入することが可能。
  • 拡張子ない場合は呼び出し元のTemplateと拡張子が一致したファイルが選ばれる。
  • サブテンプレートにもメタデータ、変数が渡され、置換される。
    • 呼び出し元/先での使用の有無は関係ないようである。
  • $title:template_sub()$ は、titleがtrueである場合に挿入する。ifのように。 * titleがarrayの場合は要素分繰り返し適用される。forのように。
  • 後述のPipesとも組み合わせ可能。$subtemplate()/uppercase$ のような形。

テンプレートの文法（Pipes）

  • テンプレート変数に対して追加の処理を行う。

  • pairs

    • mapとarrayをkeyとvalueをもったmapのarrayとする。luaのそれに相当。
    • forの中で it.key と it.value で使うのが基本か。

  • uppercase

    • 文字列のみ。大文字にする。

  • lowercase

    • 文字列のみ。小文字にする。

  • length

    • 文字列の場合文字列の長さを得る。"あいうえお"は5となった。
    • listはその長さを。 -M title=true -M title=false の場合(boolean)は0となる。

  • reverse

    • 文字列は逆順にする。"あいうえお"も"おえういあ"となった。
    • listも逆順にする。

  • first

  • last

    • listの場合最後の要素を取る。
      • ひとつの値を期待する場所はこれを付けておくと良いかも。

  • rest

  • allbutlast

  • chomp

  • nowrap

  • alpha

    • 26で割った余りがアルファベットに対応する1がa
    • 0と26が`になる（zにならない）のだが･･･？？バグか？

  • roman

    • 数字をローマ数字にする

  • left n “leftborder” “rightborder”

    • 主に表に使う想定のよう。
      • $title/right 10 "$" "@"$ のようにして使う。
      • 出力はたとえば $ hoge@ のようになる。
    • 左寄せ。nはintegerのみ（変数不可）、区切り文字は""で囲う必要がある。
      • 区切り文字はオプション。無くてもよい。
    • 文字のみ正しく動作。
      • boolean（-M titleのような）やlist(array)はNG
    • 文字数はいわゆる全角文字で2文字相当
    • 枠に入らない場合の分割の挙動はちょっとよくわからず･･･
      • (n=奇数で全角文字のみだと半角空白が入る･･･など）

  • right n “leftborder” “rightborder”

    • 右寄せ。他のルールはleftとおなじ。

  • center n “leftborder” “rightborder”

    • 中央寄せ。同上。

テンプレートの文法 （Nesting、 Breakable spaces）

  • 割愛
  • あんまり使い道が分からなかったので･･･

テンプレートを修正する

  1. include-before-body　や include-after-body を使いdocxファイルに挿入する場合、挿入箇所にファイルの内容をそのまま追加して、全体で正しいファイルとなるようにしないといけないの
    1. ファイル先頭の <?xml version="1.0" encoding="UTF-8"?> <document> <w:body> タグと末尾の対応するタグをそれぞれ削除する。
  2. テンプレート内は変数まわりはOOXMLのタグが完備されておらず（一部ある）、単に変数の文字列が代入されただけでは有効なファイルにならない。テンプレート内の$title$などの変数は以下のようなタグで囲む。デフォルトのテンプレートはabstract-titleだけこの形式になっている。
    <w:p>
      <w:pPr>
        <w:pStyle w:val="Title" />
      </w:pPr>
      <w:r>
        <w:t xml:space="preserve">$title$</w:t>
      </w:r>
    </w:p>
  • 前処理（フィルタ）で挿入部分をこのタグで囲うようにも出来るが、変数で指定された場合は出来ない
    • ･･･ので、テンプレートの修正の方が今回は良いと判断。
    • さらに、本文側も対応しようとすると、ooxmlカスタムライターをLuaで作成する必要があり･･･
      • 出来るけど無駄になりそうなのでやってない。
    • luaフィルタ内でmetaに値を入れるとテンプレートには反映されるのでmetaで統一するのはありかも？

テンプレートを使用する

  • pandoc --template=テンプレートファイル名 で使用出来る。

  • 本文に文字列があると ooxmlのタグで囲われずに埋め込まれるため、本文を空にする。

  • HTML形式の出力として扱われるため、 本文があると<p></p>として挿入されるので、本文を空にする。

    • --to plainにしてもよいが、結局はoxmlの適切なタグで囲われていないといけない
  1. 標準入力から受ける場合、 -M body=""で消去する。与える変数やメタデータは引数で与える。

echo "" | pandoc -M title=タイトル --template=template.openxml -M body="" -o template.filled.openxml

  1. ファイル入力を使うなら、本文が空のファイルを使う。
    1. 以下のようにYAMLを与えると（本文が空の）メタデータとして扱われるため良さそう（入出力共にMarkdown扱い。ymlやyamlは対応していないのでWarning出る。嫌ならmdにしておいてもよい。）

pandoc --template=template.openxml setting.yml -o template.filled.openxml

  1. 別に本文があっても-M body="" で消せば良いので、入力は本文があっても良い(YAML部分だけ設定に使う)

    　pandoc --template=template.openxml input.md -o template.filled.openxml -M body=""

ただし、表紙と本文でメタデータを変えないとタイトルが2回現れることになるので注意。

テンプレート変数の細かい話

メタデータではなく、テンプレート変数でも値をセットできるが、以下の注意点がある。

  • テンプレート変数(-V)の true / false は論理値の true / false ではなく、文字列の "true" / "false" になる。
    • -M title のように値をセットしない（あるいは true を指定する）とテンプレートには true の文字が入る
    • 複数回指定すると、list(array)扱いになる。
  • テンプレートの場合、-V と -M は-Vが指定されていれば-Vの方が優先される。
    • 補足ページで後述。
  • -M はエスケープされる。-Vはされない。
    • -M title="<hoge>" は &lt;hoge&gt; がテンプレートに埋め込まれるが、 -V title="<hoge>" はそのまま <hoge> が埋め込まれる。
    • これはYAMLがMarkdownとして読まれることに起因している、らしい。
  • また、 --toc はテンプレートに対しては無効。
    • -M toc=trueなどで明示的な値の指定が必要。

作成したファイルを確認する

  • 意図通りに値が入っているか確認しましょう。
  • Templateの場合は標準出力に出力できるので便利は便利･･･。

作成したファイルを挿入する

  • --include-before-body | -Bで本文の前に作成したファイルを追加。--include-after-body | -A で本文のあとに追加。
pandoc --from markdown input.md --to docx --include-before-body=template.filled.openxml -o out.docx --toc --reference-doc custom_reference.docx
  • カスタム表紙作成の場合、本体の方には挿入するファイルで指定した title などのメタデータを与えないようにする(当然、2回出てくる)
  • 挿入場所はTemplateの構成が参考になる（include-before-bodyとか）

補足: メタデータの強弱(1.原則)

  • Pandocはメタデータの与え方がいくつかある。
    • 入力ファイルのYAML部分
    • --metafata-fileで与えるYAMLファイル
    • オプションの -M　で与える値
    • Default file中の metadata の値
  • これらのメタデータには指定方法によって強弱（優先）関係がある。
  • メタデータ指定時の強度は以下の通り。
default fileのmetadataの値 >= -M（引数で与える）の値 > input.md の YAML > --metadata-file の YAML
  • メタデータファイルの方が弱い点は注意。
    • default fileと-Mの関係ついては別途注意事項あり（後述）
    • この関係はマニュアルから読み取れないこともない･･･

補足: メタデータの強弱(2.同強度で複数指定した場合)

  • -M 指定の場合、値はarray(list)となり、受ける側がひとつの値を取る場合は要素をつなげたものになる。
    • 入力例: -M title=xxx -M title=ooo
    • 以下のような与え方と同じ。 title: - xxx - ooo
  • 値を受ける側が複数受けられる場合は、それぞれの値が入る。
    • 例: -M author=A -M author=B Author AとAuthor Bがそれぞれ追加される
      • 目安はTemplate中で$for$ を使っている箇所。
  • metadata-fileはあとから指定したもので上書きされる。
    • 例： --metadata-file=input_meta.yml --metadata-file=input_meta2.yml のように複数指定した場合、キーの重複があるとあとから指定した値が有効（値は結合されず置き換わる。重複がなければ先に指定した値は有効。）
    • authorも上書きされた。
  • default fileもあとから指定した物で上書きされる。metadata-fileと同じ挙動。
    • 例： --defaults=default.yaml --defaults=default2.yaml のように複数指定した場合、キーの重複があるとあとから指定した値が有効（値は結合されず置き換わる。重複がなければ先に指定した値は有効。）
    • authorも上書きされた。
  • 入力ファイルのYAMLもあとから指定した物で上書きされる。
    • input.md input2.md のように複数指定した場合、キーの重複があるとあとから指定した値が有効（値は結合されず置き換わる。重複がなければ先に指定した値は有効。）
    • authorも上書きされた。

補足: メタデータの強弱(3 Defaults files と -M の衝突）

  • 指定順で挙動が異なる。--defaults より前に指定した-M は無視され、あとに指定したものは複数の-Mのケースと同じ。
    • titleのようにひとつの値を受け取る場合、
    • --defaults=default.yaml -M title=hoge は defaulthoge となる
    • -M title=hoge --defaults=default.yaml は default　となる（hogeは無視される）
    • 意図した挙動なのかは不明。
    • 外部ツールにより、引数指定順序が制御出来ない場合はこれらの例外ケースに注意。
      • ･･･なのだが、こんなことを考えるほど込み入った状況を作らない方が大事。
    • 引数末尾に指定すればdefaultsが優先されるので、設定次第で-Mを全て帳消しに出来る･･･
  • 複数指定となってlistとなった場合（文字列が結合される）は出力に下記のようなWARNINGが出るが、後者は出てこないので気づきにくい。
[WARNING] This document format requires a nonempty <title> element.
Defaulting to 'input' as the title.
To specify a title, use 'title' in metadata or --metadata title="...".

補足: どうにかして入植した値をキャンセルできないのか･･･

  • 現状既に与えたメタデータを打ち消す方法は素のPandocにはなさそう。
    • nullなどを与えても文字列になる。
    • lua filterを使えばいける。
  • -M や -V は追加していくとarrayになる。
    • arrayになると文字列期待の場合は要素をconcatされて困る･･･
    • ･･･という場合、Templateであれば、pipeのlastを使っておく方法がある。単一値を期待している場合に有効。
    • これであれば、最後に指定した値を常に使うので、複数値が入ってString期待がarrayになって困るということはなくなる。
    • 1要素であっても問題なく振る舞うので通常の用途でも問題ない。
  • デフォルトファイルに変数を与えておき、フィルタで処理を噛ませて変数を追加、テンプレートでは最後の値を使うということなどが出来る。

番外編 2段組からの切り替え

  • セクション区切りを使うことで、段組を変更出来る。
  • <w:sectPr> はドキュメント最後以外は <w:p> タグ中に入れる。
  • Templateを使う場合は以下のように修正すれば良い（例）
    • 特定箇所に挿入するだけなので、divとclassでlua filterでもいい。
$-- sectpr will be set to the last sectpr in a reference.docx, if present
$if(sectpr)$
    <w:p><w:pPr>
    <w:sectPr>
      <w:pgSz w:w="12240" w:h="15840"/>
      <w:pgMar w:top="1985" w:right="1701" w:bottom="1701" w:left="1701" w:header="720" w:footer="720" w:gutter="0"/>
      <w:cols w:num="2" w:space="720"/>
    </w:sectPr>
    </w:pPr></w:p>
$else$
    <w:sectPr />
$endif$
